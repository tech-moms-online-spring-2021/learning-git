Getting started

You can either clone an existing repository or start one from scratch

Cloning a repository

"Cloning" is how you download a repository to your local machine and get it prepped for contributing.

git clone https://github.com/tech-moms-online-spring-2021/learning-git.git

Start from scratch

  1. Log into github.com and create a new, empty repository
  2. Follow the instructions for creating a new repository on the command line ... something like this
    3. echo "# gapper-gonzo" >> README.md
    git init
    git add README.md
    git commit -m "first commit"
    git branch -M main
    git remote add origin https://github.com/sunitajennings/gapper-gonzo.git
    git push -u origin main

Both of these methods will create a new local repository that is connected to the remote repository (what you see on github.com)

Git Cheatsheet

git status What's the status of my current branch?
git checkout -b "sunita/my-first-branch" Create a new local branch called "sunita/my-first-branch", and make that your working branch
git add . Add all changes to the "staging area"
git add my_file.txt Add a specific file to the "staging area"
git commit -m "I added a new file" Commit your staged changes
git push --set-upstream origin sunita/my-first-branch Push your local changes to the remote repository. Note that you can just type git push, and git will prompt you to type out this full command.
If your push is successful, git will give you a url for creating a pull request. Create the pull request and wait for approval.
git checkout main Switch your working branch to the main branch
git fetch Tell git to look for changes (no changes will be applied, but you'll see a list of any new branches that have been created on the remote repository)
git pull Apply any changes from the remote repository to your local repository
